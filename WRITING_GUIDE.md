# 写作规范指南

## 输出要求
- **格式**: Markdown (.md)
- **编码**: UTF-8
- **每章一个独立文件**: ch{XX}_{章节名}.md（如 ch01_什么是AI_Agent.md）
- **卷级索引**: 每卷一个 vol{N}_index.md，包含卷简介和章节目录

## 写作风格
- 中文为主，技术术语保留英文原文（首次出现时标注中文）
- 代码示例使用 Python 为主，必要时补充 TypeScript
- 每章目标长度：8000-15000字（约15-25KB Markdown）
- 包含：概念讲解、代码示例、架构图（用文字描述）、最佳实践、常见陷阱

## 章节结构模板
```markdown
# 第X章：{章节名}

## 概述
{200-300字概述本章内容和目标读者}

## X.1 {第一节}
...

## X.2 {第二节}
...

## 代码示例
```python
...
```

## 最佳实践
...

## 常见陷阱
...

## 小结
...

## 延伸阅读
...
```

## 质量标准
- 概念准确，有实际参考价值
- 代码可运行或有明确说明
- 避免空洞论述，要有具体案例
- 图表用 ASCII 或 Mermaid 语法
