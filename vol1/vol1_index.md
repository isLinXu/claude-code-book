# 卷一：认知篇

> **"在编写第一个Agent之前，你首先需要理解Agent是什么、从哪里来、以及为什么要以这样的方式来编程。"**

---

## 卷简介

欢迎来到《Agent编程：从原理到生产级实践》的第一卷——**认知篇**。

在接下来的三章中，我们将一起建立对AI Agent的完整认知框架。这不是一本"快速上手"的教程——那种书太多了。我们要做的是更深的事情：**理解Agent的本质，理解它与传统软件的根本区别，理解它所代表的编程范式的变革。**

本卷适合以下读者：

- **软件开发者**：希望理解Agent编程与日常编程的本质区别
- **产品经理/架构师**：需要评估Agent技术在业务中的适用性
- **AI研究者**：想从工程角度理解Agent的实现思路
- **技术决策者**：需要为团队制定Agent技术路线

即使你已经有Agent开发经验，我也建议你通读本卷。很多时候，"知道怎么做"和"理解为什么这样做"之间的差距，正是普通工程师和优秀架构师的分水岭。

```
第1章（概念基础）
    │
    ├──→ 第2章（历史脉络）
    │
    └──→ 第3章（编程范式）
              │
```

- **第1章**是基础，建议必读

## 阅读建议

1. **不要跳过代码示例**——即使你觉得已经理解了概念，代码中往往藏着细节
2. **"常见陷阱"部分值得反复阅读**——那些是前人踩过的坑
3. **每章末尾的延伸阅读**——按兴趣选择，不必全部读完
4. **带着问题阅读**——如果你正在考虑某个具体的Agent项目，边读边想会更有收获
